# BuyPadi Frontend

## Design Context

### Users
BuyPadi internal admins / operations staff signing into the secure ops console (passwordless magic-link auth). Desktop-first, used occasionally; they want fast, trustworthy access. The broader app also serves Nigerian buyers using an inspect-before-you-pay logistics service.

### Brand Personality
Trusted, grounded, precise. Premium but practical — not flashy. The product's whole promise is verification and safety, so interfaces should feel calm, credible, and deliberate.

### Aesthetic Direction
- **Theme: light / off-white** (`surface` #F5F7FA). NOT dark-green page backgrounds.
- Brand: dark green `primary` #063B27, `lime` #8DC342 accent (used sparingly). Tokens in `app/globals.css`.
- Fonts: Plus Jakarta Sans (display) + Inter (body) — kept for whole-app consistency even though they'd otherwise be avoided.
- Quiet, meaningful background texture: the **"Scattered proof"** pattern (faint dot field + brand line-motifs: magnifier=inspect, shield=verified, pin=anywhere, parcel=deliver + a focal "trust pulse" ring), dark motifs at low opacity on off-white — same language as the 404 page.
- Editorial, type-led, left-aligned, asymmetric. Restraint over decoration.

### Anti-references (what it must NOT look like)
- Dark-green full-bleed backgrounds for utility pages.
- Two-tone split "marketing card" login layouts.
- Decorative icon-in-rounded-square "seals" above headings; shield/lock clichés as UI chrome.
- Glassmorphism, gradient text, side-stripe accent borders, everything-in-a-card.
- Generic centered card floating on dead empty space.

### Design Principles
1. Off-white surface with subtle, meaningful pattern — never empty, never loud.
2. Type-led hierarchy; left-aligned and asymmetric, not centered-card clichés.
3. Restraint: no decorative icon seals, no glass, no gradient text, no stripe borders.
4. One accent (lime), used rarely, so it carries weight. Dark green for primary actions.
5. Consistent with the 404 and the rest of the BuyPadi app.
